# 模式：project —— Portfolio 專案評估

6 維評分（1–5）：

| 維度 | 權重 | 5 = ... | 1 = ... |
|------|------|---------|---------|
| 對目標角色的訊號 | 25% | 直接展示 JD 所列 skill | 無關 |
| 獨特性 | 20% | 沒人做過 | 每個人都有 |
| 可 demo 程度 | 20% | 2 分鐘內 live demo | 只有 code，不可視 |
| 指標潛力 | 15% | 明確指標（latency、cost、accuracy）| 無法量測 |
| 到 MVP 的時間 | 10% | 1 週 | 3 個月以上 |
| STAR 故事潛力 | 10% | 有 trade-off 的豐富故事 | 只有實作 |

## 「Interview Pack」需求

每個通過的專案都要有：
1. **One-pager：** 產品 + 架構 + 指標 + 評估計畫
2. **Demo：** Live URL 或 2 分鐘錄製 walkthrough
3. **Postmortem：** 什麼 work、什麼不 work、怎麼 mitigate

## 80/20 計畫

- 第 1 週 → MVP + 核心指標
- 第 2 週 → polish + interview pack

## 裁決

- **做** → 逐週 milestone 計畫
- **SKIP** → 為什麼不做，以及改做什麼
- **PIVOT 到 [替代方案]** → 更有影響力的變形
